- Learn AI-assisted engineering workflows
- Practice TypeScript, React, and Node.js

- Understand every implementation decision

## Tech Stack

- TypeScript
- Node.js

## Rules for Claude

- Explain plans before coding
- Make small focused changes
- Avoid unnecessary complexity
- Keep code readable
- Ask before large refactors
- Add tests where appropriate
- Explain architectural decisions

- Do not commit changes unless I ask

- explain the plan
- list assumptions
- mention possible tradeoffs
- wait for my approval before editing files

Prioritize:
- readability
- simplicity
- extensibility
- small focused changes

Avoid:
- unnecessary abstractions
- premature optimization
- overly complex patterns
- large changes without explanation
